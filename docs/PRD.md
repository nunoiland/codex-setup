# PRD Guide

Use a PRD as the source of truth for non-trivial work in this repo.

## Required sections
- Goal
- Scope
- Non-goals
- Context
- Constraints
- Acceptance criteria
- Edge cases
- Platform target
- Validation commands
- Done when

## Also required for unattended or delivery work
- Delivery mode
- Reporting mode
- Secret profile
- Human handoff

## Repo-specific rules
- Start from `PRD/_template.md`.
- Keep platform target explicit.
- Keep validation commands concrete and runnable.
- Call out anything needing human approval.
- Prefer narrow scope over aspirational prose.
